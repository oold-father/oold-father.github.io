12.1.3. SSH设置

# ssh登录时，登录ip被会反向解析为域名，导致ssh登录缓慢
sed -i "s/#UseDNS yes/UseDNS no/" /etc/ssh/sshd_config
sed -i "s/GSSAPIAuthentication yes/GSSAPIAuthentication no/" /etc/ssh/sshd_config
sed -i "s/GSSAPICleanupCredentials yes/GSSAPICleanupCredentials no/" /etc/ssh/sshd_config
sed -i "s/#MaxAuthTries 6/MaxAuthTries 10/" /etc/ssh/sshd_config
# server每隔30秒发送一次请求给client，然后client响应，从而保持连接
sed -i "s/#ClientAliveInterval 0/ClientAliveInterval 30/" /etc/ssh/sshd_config
# server发出请求后，客户端没有响应得次数达到3，就自动断开连接，正常情况下，client不会不响应
sed -i "s/#ClientAliveCountMax 3/ClientAliveCountMax 10/" /etc/ssh/sshd_config

12.1.4. 添加swap虚拟内存

交换文件
# 建立交换文件
dd if=/dev/zero of=/swapfile bs=1M count=512
# 为交换文件设置权限：（交换文件全局可读是一个巨大的本地漏洞）
chmod 600 /swapfile
# 创建好交换文件后，将其格式化：
mkswap /swapfile
# 启用交换文件：
swapon /swapfile
# 最后，编辑 /etc/fstab， 在其中添加如下的一行：
cat << EOF >/etc/fstab
/swapfile none swap defaults 0 0
EOF
systemctl daemon-reload

13.3. MQTT-paho

13.3.1. python-paho常见问题以及解决方案

client id问题

mosqutto服务器有限制，每一个客户端连接都需要指定自己的client id（paho库有一个默认id），当一个客户端连接上mosqutto服务器时，mosqutto会把之前带有该id的客户端下线，再与新的客户端建立连接。换句话说，如果要保证一个应用内有多个mosqutto客户端保持连接，就要为每个mosqutto客户端设置不同的id

解决办法:
每次使用mosqutto客户端时都按照功能和用途指定详细的id

loop_forever问题

当mosqutto客户端消费者使用loop_forever监听队列时，可能会导致其他的使用loop_start监听队列的消费者离线。举个例子，有一个消费者A使用loop_start的方式后台监听队列，再有一个消费者B使用loop_forever监听队列，消费者A可能会被离线，原因未知

解决办法:
如果一个应用内需要同时启动多个监听，全部都使用loop_start来监听消息队列。如果需要保证程序不退出，采用在最后一个loop_start监听后添加无限循环来保证

publish_single问题

paho提供了只发送单条消息的方法 publish_single ，该方法会自动创建一个连接并发送消息，最终关闭连接。在libthcl-python库中使用MqttPub对这一方法进行了封装，使用 publish_single 会出现两个客户端被启动，只有一个客户端停止的情况。

解决方法: 已经修复。目前代码里采用的是主动调用 disconnection 方法关闭连接，代码如下:

pub = MqttPub()
pub.init()
pub.publish_single('test', 'test', 0)
pub.mqtt_client.disconnect()

16.4. python共享文件服务

使用python的http模块可以快速搭建起一个http文件服务器, 只需要在命令行使用 python -m http.server 8000 就可以将当前文件夹作为http服务的目录。基于这一点我们可以方便的进行文件共享。

由于共享次数太多，每次需要手动输入指令, 并且需要占用一个终端，所以经想到了利用 systemd 来管理服务。

添加service文件
mkdir /share

sudo -i

cat << EOF >/usr/lib/systemd/system/http-share.service
[Unit]
Description=python http server
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=yinxin
WorkingDirectory=/share
ExecStart=/usr/bin/python -m http.server 2345

[Install]
WantedBy=multi-user.target
EOF
启动服务
systemctl enable http-share.service
systemctl start http-share.service
systemctl status http-share.service

16.5. python多语言支持

python多语言支持使用gettext, 主要分为两个部分: 根据po文件生成mo文件，使用mo文件实现多语言。

16.5.1. 生成po文件

po文件可以使用 xgettext -d foo -o locales/foo.pot foo.py 从代码生成

foo.py的写法如下

import  gettext
_ = gettext.gettext

_("你好")

生成的pot文件如下

# SOME DESCRIPTIVE TITLE.
# Copyright (C) YEAR THE PACKAGE'S COPYRIGHT HOLDER
# This file is distributed under the same license as the PACKAGE package.
# FIRST AUTHOR <EMAIL@ADDRESS>, YEAR.
#
#, fuzzy
msgid ""
msgstr ""
"Project-Id-Version: PACKAGE VERSION\n"
"Report-Msgid-Bugs-To: \n"
"POT-Creation-Date: 2020-06-17 16:54+0800\n"
"PO-Revision-Date: YEAR-MO-DA HO:MI+ZONE\n"
"Last-Translator: FULL NAME <EMAIL@ADDRESS>\n"
"Language-Team: LANGUAGE <LL@li.org>\n"
"Language: \n"
"MIME-Version: 1.0\n"
"Content-Type: text/plain; charset=UTF-8\n"
"Content-Transfer-Encoding: 8bit\n"

msgid "你好"
msgstr ""

添加翻译

msgid "你好"
msgstr "hello"

直接将pot重名为po即可

可以使用这种方式生成po文件，也可以自己根据文件格式手写。 如果需要使用 Content-Type 这样的类似于请问头的属性，最前面的 'msgid "" msgstr ""' 需要保留

16.5.2. 生成mo文件

使用 msgfmt -o locale/foo/LC_MESSAGES/foo.mo foo.po 生成mo文件

mo文件的目录层级是固定的，其中 LC_MESSAGES 是固定文件夹名，如果需要添加多个语言的支持，则在 locale 下添加 bar/LC_MESSAGES/bar.mo 即可

16.5.3. 使用mo文件完成多语言支持

import  gettext

lang_translations = gettext.translation('foo', localedir='locale', languages=['foo'])
lang_translations.install()
# define _ shortcut for translations
_ = lang_translations.gettext

# mark a string translatable
print(_("你好"))

这条命令会通过代理下载pip包

配置文件下

maxmemory-policy volatile-lru
    volatile-lru:从已设置过期时间的数据集(server.db[i].expires)中挑选最近最少使用的数据淘汰
    volatile-ttl:从已设置过期时间的数据集(server.db[i].expires)中挑选将要过期的数据淘汰
    volatile-random:从已设置过期时间的数据集(server.db[i].expires)中任意选择数据淘汰
    allkeys-lru:从数据集(server.db[i].dict)中挑选最近最少使用的数据淘汰
    allkeys-random:从数据集(server.db[i].dict)中任意选择数据淘汰
    no-enviction(驱逐):禁止驱逐数据,新写入操作会报错
    ps:如果没有设置 expire 的key, 不满足先决条件(prerequisites); 那么 volatile-lru, volatile-random 和volatile-ttl 策略的行为, 和no-enviction(不删除)基本一致

开启bbr
cat << EOF > install.sh
